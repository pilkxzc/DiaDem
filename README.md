# DiaDem — Децентралізована Соціальна Мережа

> Блокчейн + IPFS + P2P. Без серверів. Без алгоритмів. Без реклами. Без посередників.

**DiaDem** — це пірінгова соціальна мережа, що працює прямо у вашому браузері. Ваші дані зберігаються на блокчейні та IPFS (контентно-адресоване сховище). Кожна вкладка браузера — це повноцінний вузол мережі.

## Особливості

- **Власний блокчейн** — повністю децентралізований консенсус
- **P2P мережа** — автоматичне виявлення пірів через WebRTC
- **ECDSA криптографія** — цифрові підписи для автентифікації
- **Контентно-адресоване сховище (CAS)** — дані зберігаються за хешем
- **100% клієнтський** — весь код працює у браузері
- **Без серверів** — сигнальний сервер тільки для початкового з'єднання пірів
- **Мультимовність** — підтримка інтернаціоналізації (i18n)


## Встановлення

```bash
# Клонувати репозиторій
git clone https://github.com/YOUR_USERNAME/DiaDem.git
cd DiaDem

# Встановити залежності
npm install

# Запустити сервер
npm start
```

Сервер запуститься на `http://localhost:3000`

## Режими запуску

```bash
# Продакшн
npm start

# Розробка (з CORS та без кешу)
npm run dev

# Вказати порт
node server.js --port 8080
```

## API

| Ендпоінт        | Опис                                  |
|------------------|---------------------------------------|
| `GET /api/info`  | Інформація про вузол (версія, піри)   |
| `GET /api/peers` | Список підключених пірів              |
| `WS /signal`     | WebSocket сигналінг для WebRTC        |

## Технології

- **JavaScript (ES Modules)** — без фреймворків, чистий JS
- **WebRTC** — пряме P2P з'єднання між браузерами
- **WebSocket** — сигнальний сервер для виявлення пірів
- **Web Crypto API** — ECDSA підписи та хешування
- **Node.js** — мінімальний HTTP/WS сервер

## Як це працює

1. Користувач відкриває DiaDem у браузері
2. Генерується криптографічний гаманець (ECDSA ключова пара)
3. Браузер підключається до сигнального сервера
4. Через WebRTC встановлюються прямі P2P з'єднання з іншими вузлами
5. Дані (пости, профілі) зберігаються на блокчейні та в CAS
6. Кожен вузол має повну копію ланцюга
