How DiaDem Works

Own Blockchain

Custom Proof-of-Stake blockchain with DDM token. Every transaction — posts, follows, likes, transfers — is a signed transaction on-chain.

IPFS Content Storage

IPFS-compatible CAS protocol. Every piece of data is identified by its hash. Content is published to the IPFS network and replicated across peers.

WebRTC P2P Network

Direct peer-to-peer connections via WebRTC. Automatic peer discovery via signaling server. BroadcastChannel for local tabs. No manual setup needed.

How P2P Works in DiaDem

Signaling Server — WebSocket relay for automatic peer discovery. No manual offer/answer exchange needed.
BroadcastChannel — Automatic sync between tabs in the same browser. Zero config.
WebRTC DataChannel — Direct peer-to-peer connections across devices. Uses public STUN servers for NAT traversal.
IPFS Bridge — Content-addressable data is also published to the IPFS network when a local node is available. Retrieval via public gateways.
Protocol — HELLO → SYNC_REQUEST → SYNC_RESPONSE → NEW_BLOCK / NEW_TX
Data Replication — CAS objects (WANT/HAVE/BLOCK) automatically sync between connected peers + IPFS.

About DiaDem

Version: 0.1.0 (Testnet)
Architecture: Blockchain + CAS + P2P
Consensus: Proof of Stake
Token: DDM (100M total supply)
Crypto: ECDSA P-256 + SHA-256
Network: WebRTC + BroadcastChannel + Signaling
Storage: Content-Addressable (IPFS-compatible)
IPFS: Gateway reads + local node publishing
Database: None (fully on-chain + CAS)
Server: Minimal signaling relay (no data stored)
Every browser tab runs a complete DiaDem node with its own
blockchain, CAS storage, and P2P networking capability.
